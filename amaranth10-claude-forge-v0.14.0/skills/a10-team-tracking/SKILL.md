# 팀 트래킹 (a10-team-tracking)

ST Step 7에서 신설된 6 관리자 폴더(`team-tracking/`)에 부하 직원 트래킹 노트를 표준 골격으로 작성하는 스킬. 2계층 구조 (`{관리자}/{대상}/{YYYYMMDD}-{주제}.md`) — Q3 (B) 결정.

## 목적

- 관리자(Cell 리더·Unit장·본부장)가 부하 직원 성과·이슈·평가·코칭 메모를 표준 골격으로 작성
- Step 7 6 관리자 폴더 패턴 일관 + G8 a10-people-index 연계로 사번/이름 자동 채움
- 계층적 공유 권한 (Cell 리더 → Unit장 → 정현수 전무) 일관 운영

## SSoT 경로 (외부 참조, 보강 A)

| 항목 | 경로 |
|------|------|
| 6 관리자 폴더 (절대) | `~/Workspace/douzone-forge/team-tracking/` |
| douzone-forge 마운트 시 (상대) | `team-tracking/` |
| 사번 SSoT (G8 연계) | `~/Workspace/douzone-forge/_CLAUDE/프로세스/사용자ID-매핑.md` |

⚠️ 본 스킬은 **douzone-forge 워크스페이스 마운트를 가정**. 다른 워크스페이스에서 호출 시 사번 매핑 SSoT(G8 a10-people-index)와 동일 워크스페이스 동시 마운트 필요.

## 입력 / 출력 (Q3 (B) 2계층)

### 입력

- 관리자 사번 또는 이름 (예: `B3505` 또는 `유지수`)
- 트래킹 대상 사번 또는 이름 (예: `B6276` 또는 `손예진`)
- 일자 (예: `20260429`)
- 주제 (예: `주간트래킹`, `1on1`, `월간평가`)

### 출력 경로 (2계층)

```
team-tracking/{관리자-{이름}_{사번}}/{대상-{이름}_{사번}}/{YYYYMMDD}-{주제}.md
```

예시: `team-tracking/Cell리더-유지수_B3505/손예진_B6276/20260429-주간트래킹.md`

### 노트 표준 골격

```markdown
---
관리자: {이름} ({사번})
트래킹 대상: {이름} ({사번})
일자: YYYY-MM-DD
주제: {주간트래킹 | 1on1 | 월간평가 | 기타}
---

# {YYYY-MM-DD} {대상 이름} {주제}

## 사실 (Facts)
- 진행 중 작업: ...
- 완료된 산출: ...
- 일정 진척: ...

## 평가 (Assessment)
- 성과: ...
- 부담/이슈: ...
- 협업 관계: ...

## 액션 (Actions)
- 즉시: ...
- 1주 내: ...
- 다음 점검 시: ...

## 다음 점검 일자
- {YYYY-MM-DD}
```

## G8 a10-people-index 연계

본 스킬은 입력으로 사번 또는 이름을 받으면 **G8 a10-people-index** 를 호출하여:
- 관리자/대상의 정확 사번 + 이름 + 소속 (Cell) 자동 조회
- 폴더 경로 자동 생성 (`{역할}-{이름}_{사번}/`)
- 노트 frontmatter 자동 채움

## R&R 권한 검증

| 작성자 | 트래킹 권한 (대상) | 비권한 |
|------|------|------|
| Cell 리더 4명 (신무광·유지수·김경엽·이정미) | 본인 Cell 멤버 (5~6명) | 다른 Cell 멤버 |
| Unit장 (차민수 B2772) | SBUnit 22명 전원 + Cell 리더 4명 | 협업 외부 인원 |
| 정현수 전무 | Unit장 트래킹 결과 열람 | 직접 트래킹 작성 (협업) |

권한 외 영역 미접근 — 권한은 **사회적 합의 + 커밋 책임** 으로 통제 (git 차원 강제 불가).
